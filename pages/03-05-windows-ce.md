게임 회사에 취업하기 망설여 졌던 부분은 Windows CE라는 저에게는 생소한 운영체제에서 돌아가는 게임을 만들어야 된다는 것이었습니다. 하지만 Windows API를 이용해 어플리케이션을 개발하고 SDL(Simple DirectMedia Layer)를 이용해 게임을 개발하는데 지장이 없었습니다. DirectDraw 직접 사용하지 않아도 SDL이 DirectX나 OpenGL을 wrapping 하고 있어서 개발하는데 편리하고 개발 속도도 상대적으로 빨랐습니다. 직접 회사에 입사해서 개발해 보면서 저의 걱정은 사라졌습니다.

게다가 그 당시 Windows Mobile OS는 Windows CE 기반이었기 때문에 윈도 폰에서도 돌아갈 수 있었습니다. 그러다가 애플의 스티브 잡스가 아이폰을 내놓으면서 게임 업계가 폭풍을 맞을 정도의 충격이 컸습니다. 그렇다고 해서 제가 주로 개발했던 Windows CE 운영체제에서 돌아가는 게임을 개발한 것이 후회가 되지 않았습니다. 그 이후로도 쭈욱 제가 개발자로서 성장하는데 좋은 경험이었습니다.

지금도 SDL 공식 홈페이지(https://www.libsdl.org/) 가 있고 꾸준히 업데이트 되는 것을 보면 대단하다고 생각합니다.
