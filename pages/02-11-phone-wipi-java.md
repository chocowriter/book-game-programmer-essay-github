대학교 4학년이 되어서 졸업작품은 피쳐폰에서 돌아가는 게임을 개발하기로 하였습니다. 국내 표준이 된 WIPI가 개발하기 수월해 보였습니다. C++은 아직 저에게 어려운 프로그래밍 언어였습니다. 그래서 Eclipse IDE에서 Java 언어로 게임을 개발하였습니다.

게임 기획은 슈퍼 마리오와 유사하지만 좀 더 매니악하고 레벨이 복잡합니다. 하지만 재미는 좀 없었습니다. 레벨 디자인이 개발하기 어렵기도 하고 개발 진척도가 많이 떨어졌습니다.

다행히도 당시 WIPI Java로 게임 개발하는 서적이 시중에 하나 있더군요. 그래서 책에 있는 샘플 코드를 바탕으로 수정을 많이 하였습니다. 책 한권을 다 보고나자 기획에 맞추어 코드 수정을 할 수 있었습니다.

그러나 큰 문제가 있었습니다. 맵 에디터를 개발할 시간이 부족했습니다. 그래서 레벨디자인도 속도가 많이 더디었습니다. 기획 문서와 스프라이트 이미지로만 게임 개발을 해야 했기 때문에 손이 많이 갔습니다. 개발 속도가 나가지 않으니 재미에 대한 검증도 이뤄지지 않아서 많이 아쉬운 졸업 작품 프로젝트였습니다.

졸업 작품에 대한 논문을 쓰고 중국 교환학생들을 대상으로 소스코드 설명을 하였습니다. .java 코드가 무려 3개 밖에 되지 않았습니다. 그래서 하나의 소스파일의 길이가 1000 단위를 넘겼습니다. 지금 생각해 봐도 참으로 부끄럽습니다. 그 당시에는 객체 지향 프로그래밍에 대한 개념이 장착되지 않았습니다. 돌아가는 코드를 바탕으로 코드 개선하는 것조차 어려웠습니다.
