개발 생산성을 위해 다른 개발자가 짠 코드로 개발하다 보면, 코드 분석은 잘 될지 몰라도 개발 실력은 그렇게 좋아지지 않았습니다.

유니티 에셋 스토어를 자주 사용합니다. 에셋 스토어에도 없다면 GitHub에서 검색해 봅니다. 마지막으로 Claude에게 질문해도 맞는 답변을 얻을 수 없다면, 저 스스로 개발해야 합니다.

구글링 => 유니티 에셋 스토어 => GitHub => Claude를 거치면 대부분의 문제를 해결할 수 있었습니다.

이런 개발 방법은 빠르게 개발할 수 있어서 생산성이 좋은 반면, 하나부터 열까지 한땀한땀 구현하다 보면 개발 실력을 쌓는 데 크게 도움이 됩니다. 답안지가 이미 있지만, 보지 않고 스스로 푸는 것과 유사합니다. 그러다 보니 스스로 생각하는 힘이 많이 떨어지게 되었습니다. Claude가 코드와 함께 답변을 워낙 잘 해주기 때문입니다.
